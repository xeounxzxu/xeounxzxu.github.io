Hugo를 사용해서 GitHub Pages 기반 블로그를 열었습니다.

이 글은 블로그의 첫 번째 포스트이자, 앞으로 어떤 내용을 기록할지 정리해 두는 시작점입니다. 복잡한 CMS 대신 정적인 구조를 선택한 이유는 단순합니다. 글쓰기에 집중하기 쉽고, 저장소에서 이력 관리도 명확하기 때문입니다.

## 이 블로그에서 다룰 내용

- 개발하면서 정리해 둘 만한 개념
- 트러블슈팅 과정과 해결 방법
- 새로 적용해 본 도구나 프레임워크
- 짧더라도 나중에 다시 참고할 기록

## 현재 구성

이 블로그는 다음 조합으로 운영합니다.

- 정적 사이트 생성: `Hugo`
- 배포: `GitHub Pages`
- 글 작성 방식: Markdown

새 글은 `content/posts/` 아래에 Markdown 파일을 추가하는 방식으로 작성합니다.

## 첫 글을 남기며

완성된 블로그를 한 번에 만드는 것보다, 짧은 글이라도 꾸준히 쌓는 편이 더 중요하다고 생각합니다. 그래서 첫 포스트는 구조를 설명하는 간단한 글로 시작했습니다.

다음 글부터는 실제로 작업하면서 마주친 내용들을 중심으로 기록해 보겠습니다.
