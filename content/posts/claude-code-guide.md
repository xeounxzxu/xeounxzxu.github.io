AI가 코드를 짜주는 시대입니다. 하지만 같은 도구를 써도 결과는 완전히 다릅니다.

클로드 코드를 만든 사람들과 해커톤 우승자들은 어떻게 활용하고 있을까요? 그들이 공통적으로 사용하는 7가지 핵심 전략을 정리했습니다.

## 핵심 요약

| 전략 | 핵심 내용 | 기대 효과 |
| --- | --- | --- |
| 병렬 작업 | 여러 터미널 동시 실행 | 작업 속도 향상 |
| `CLAUDE.md` | 프로젝트 기억 저장 | 반복 실수 방지 |
| 플랜 모드 | 설계 먼저 | 구조 안정성 향상 |
| 서브 에이전트 | 역할 분리 | 전문성 향상 |
| 검증 루프 | 결과 증명 요구 | 품질 향상 |
| 커스텀 스킬 | 반복 작업 자동화 | 생산성 향상 |
| 훅 자동화 | 실수 사전 차단 | 안정성 향상 |

## 1. 병렬 작업: 한 명이 아니라 팀처럼 일하기

터미널을 하나만 쓰지 마세요.

```text
terminal 1 -> API 구현
terminal 2 -> 테스트 작성
terminal 3 -> 리팩토링
```

이렇게 역할을 나누면 작업 속도가 체감상 2배 이상 빨라집니다.

## 2. `CLAUDE.md`: 프로젝트의 기억을 심어라

`CLAUDE.md`는 단순 문서가 아닙니다. AI의 장기 기억 시스템에 가깝습니다.

예시는 아래와 같습니다.

```md
# CLAUDE.md
- 이 프로젝트는 TypeScript strict mode 사용
- API 응답 형식은 항상 camelCase
- 이전에 null 처리 버그 발생 -> 항상 optional 체크
```

실수를 기록할수록 AI는 점점 더 정확하게 같은 프로젝트 문맥을 따라옵니다.

## 3. 플랜 모드: 먼저 설계하고 코딩하라

바로 코드부터 짜지 마세요.

먼저 이렇게 물어보는 편이 낫습니다.

> 이 기능을 시니어 엔지니어 관점에서 설계해줘

그리고 한 번 더 확인합니다.

> 이 설계의 문제점은 뭐야?

이 과정을 거치면 구현 전에 구조적 문제를 상당수 걷어낼 수 있습니다.

## 4. 서브 에이전트: 전문가에게 맡겨라

하나의 AI에게 모든 걸 시키지 마세요. 역할을 나누는 편이 결과가 더 깊어집니다.

- Backend 전문가
- 테스트 전문가
- 보안 전문가

이렇게 책임을 분리하면 같은 주제라도 훨씬 정교한 결과를 얻기 쉽습니다.

## 5. 검증 루프: 증명하게 만들어라

가장 중요한 습관입니다.

나쁜 질문:

> 이거 맞아?

좋은 질문:

> 테스트로 이게 동작하는 걸 증명해줘

답변을 받는 것보다 결과를 검증시키는 쪽이 품질을 크게 끌어올립니다.

## 6. 커스텀 스킬: 반복은 자동화하라

반복 작업은 자산입니다.

예를 들면 이런 것들입니다.

- `/test`
- `/doc`
- `/refactor`

기준은 간단합니다. 두 번 이상 반복하면 자동화 후보로 보는 편이 좋습니다.

## 7. 훅(Hooks): 실수를 자동으로 막아라

코드 작성보다 중요한 건 실수 방지 시스템입니다.

예시는 아래와 같습니다.

- `pre-commit` -> lint 실행
- `post-tool` -> 코드 포맷 정리

작은 실수를 사람 손으로 계속 잡는 대신, 시스템이 먼저 막도록 만드는 방식입니다.

## 결론

고수들은 AI를 단순한 도구로 쓰지 않습니다. 시스템으로 만듭니다.

오늘부터는 딱 두 가지만 먼저 적용해도 충분합니다.

- `CLAUDE.md` 만들기
- 플랜 모드 사용

이 두 가지부터 시작해도 작업 방식이 빠르게 달라질 수 있습니다.
